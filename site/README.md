# 葬AI Web4 — Frontend

Next.js 14 static site for the 葬AI knowledge graph. Renders an interactive Cytoscape graph, article pages, and leaderboards.

## Prerequisites

- Node.js 18+
- Data files in `../web-data/` (generated by the Python pipeline)

## Development

```bash
npm install
npm run dev          # Starts dev server at http://localhost:3000
```

The `prebuild.sh` script automatically copies `../web-data/` → `public/data/` before each dev/build.

## Build

```bash
npm run build        # Static export to out/
```

Output is a pure static site (`next export`), deployable to Vercel / Netlify / Cloudflare Pages.

## Deploy

```bash
npm run build
npm run deploy
```

- `npm run deploy` is the recommended Cloudflare Pages deploy path for this repo and shell environment.
- `npm run deploy` now runs the root data pipeline build and `scripts/kg_review_gate.py` before building and uploading, so stale holistic graph reviews or missing article entities block deployment.
- The script explicitly unsets local proxy variables before calling `wrangler pages deploy out --project-name funeral-ai-web4`, because proxy settings on this machine can break wrangler OAuth refresh against `dash.cloudflare.com`.
- Wrangler auth for this macOS environment lives in `~/Library/Preferences/.wrangler/config/default.toml`, not in repo `.env` files.
- Keep `npm run deploy:raw` only for cases where you intentionally want the original wrangler command without clearing proxy variables.

## How Data Gets to the Frontend

1. Python pipeline generates JSON files in `web-data/`
2. `prebuild.sh` copies them to `site/public/data/`
3. SSG pages read `public/data/` at build time (`lib/data.ts`)
4. Graph page fetches `/data/graph-view.json` client-side

## Frontend Layout Contracts

Frontend maintainability rules live in `../docs/frontend-layout-contracts.md`.

Read that file before touching:

- page shells / standard containers
- navbar or viewport offsets
- leaderboard alignment
- primitive layout behavior

## Adding a Node Type

1. `src/lib/types.ts` — Add to `NodeType` union
2. `src/lib/constants.ts` — Add entry to `NODE_TYPE_REGISTRY`

Everything else (colors, labels, badges, legend, controls, filters) derives automatically.
